## Core method (verified SCT 2026-07-04, 118 ROs)

1. **Pull closed warranty ROs** via OpenAPI `/repair-orders:search` (tekion_client.get_token, dealer_id e.g. st/876): status IN CLOSED,INVOICED, close window = the MTD/period range, filter to ROs carrying a WARRANTY pay type.
2. **The closer = `modifiedByUserId`** on the closed RO — the last user who touched it at close time. There is NO explicit "closed by" field. Validate: modifiedTime should ≈ CLOSED_TIME (matched 116/118 on SCT July MTD; the 2 misses were System auto-closes).
3. **Resolve users** via `GET /openapi/v4.0.0/users/{id}` → `userNameDetails.completeNames[DISPLAY_NAME]` + `userRoleDetails.primaryRole.persona`. Signal is CLEAN when resolved users have persona **WARRANTY_CLERK** — state this validation in the report. Unresolvable/system ids → "System (auto-close)" bucket (normal, keep it visible).
4. **Warranty $** = per-RO fan-out to `ro-invoices`, sum invoiceAmount for the WARRANTY payType. Total RO $ = sum across all payTypes. **ALL AMOUNTS ARE CENTS — /100.**
5. Output: summary table (admin | ROs closed | warranty $ | total RO $) + RO-level CSV to `/home/itadmin/tekion-reports/data/<STORE>-Warranty-Admin-Closings-<period>.csv` (grouped by admin, sorted $ desc, open/close timestamps).
